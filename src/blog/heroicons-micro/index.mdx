我们刚刚发布了 [Heroicons v2.1](https://heroicons.com)，其中包含全新的微型风格——一整套近三百个设计用于更紧凑、更高密度 UI 的 16&times;16 图标。

<p>
  <Link href="https://heroicons.com">
    <Image src={card} alt="Heroicons Micro" />
  </Link>
</p>

我们一直想做这样一个尺寸的图标集合，但直到今年早些时候开始制作 [一个新的 React UI 套件项目](https://twitter.com/steveschoger/status/1725246591269769363) 时，我们才急需这些图标，因此决定狠下心来设计它们。

在 Catalyst（我们的新 UI 套件）中，我们努力在给事物足够的呼吸空间与保持足够密集之间取得平衡，以确保使用该套件构建的应用程序实际上感觉高效。

我们使用了大量的 14px 字体，这在 UI 中看起来很好，但当我们尝试将现有的 20×20 图标整合进来时，发现它们显得有点过大且不够平衡。

<Image alt="Heroicons Mini 与用于下拉菜单的 Heroicons Micro 比较" src={miniVsMicroDropdown} />

大多数人可能只会将图标缩小并继续前进，但我们不是大多数人，虽然这也许是好事，也可能是坏事。

图标设计为将要使用的确切尺寸时，通常看起来更清晰，所以我们开始了从头重新绘制每个图标的过程，仔细地逐个减少细节，以确保它们在预期尺寸下呈现出良好的清晰度。

<Image alt="为 Heroicons Micro 调整的细节示例" src={miniVsMicroDetails} />

大约一个月后，我们有了一套全新的 288 个图标，精心设计用于像 Catalyst 这样的高密度界面。

<div className="my-12">
  <div className="not-prose grid grid-cols-[repeat(auto-fill,1rem)] justify-between gap-x-12 gap-y-8">
    {Object.entries(icons)
      .sort(([a], [b]) => a.localeCompare(b))
      .map(([name, Icon]) => (
        <Icon key={name} className="h-4 w-4 fill-slate-950 dark:fill-white" />
      ))}
  </div>
</div>

你可以在 [Heroicons 网站](https://heroicons.com) 查看所有新图标，使用我们官方的 [Heroicons Figma 库](https://www.figma.com/community/file/1143911270904274171) 开始设计，并通过我们的 [React 和 Vue 库](https://github.com/tailwindlabs/heroicons) 将它们添加到你的项目中。
