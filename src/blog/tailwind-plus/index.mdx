我们刚完成了一项巨大的品牌重塑项目，将之前被称为 Tailwind UI 的产品转变为 [Tailwind Plus](https://tailwindcss.com/plus)。

在您下结论之前，这里有一些内容是 _不会_ 改变的：

- **没有订阅定价** — 这仍然是一次性购买并享有终身访问权，跟以前一样。
- **没有升级费用** — 如果您已经拥有 Tailwind UI 全访问许可证，则现在您拥有 Tailwind Plus 许可证。新客户的价格也不会改变。
- **所有相同的内容** — 您可以访问所有相同的内容，包括我们所有的组件、模板和 Catalyst UI 套件。

那么为什么要进行更改呢？简单来说，我们想为我们的社区做很多事情，但这些事情感觉不适合放在 Tailwind UI 的框框里。

以下是我们为 Tailwind Plus 探索的一些想法：

- **Tailwind Play 账户** — 这样您可以保存实验，更容易找回旧创意，甚至可以在不丢失原始 URL 的情况下更新现有的 Plays。
- **自定义编辑器主题** — 人们一直在问我们关于我们使用的自定义语法主题，而 Tailwind Plus 似乎是让这些主题可供大家使用的一个好方式。
- **调色板** — Tailwind CSS 默认色彩调色板的替代方案，由专家设计师为您精心策划。
- **排版主题** — 更多的排版样式，类似于我们排版插件的样式，但具有不同的风格选择。
- **建立更强大的私密社区** — Tailwind 被数以百万计的人使用，这令人难以置信，但也意味着公共渠道中有很多噪音。我们想要建立一个优秀的社区，让您可以与其他 UI 工程师进行高质量的前端开发对话，并与 Tailwind CSS 团队直接联系。
- **提前访问beta文档** — 在新功能公开发布之前，获取文档的访问权限，并帮助我们在功能稳定之前进行完善。
- **提前注册线下活动** — 如我们举办的 Tailwind Connect 见面会，您可以与 Tailwind 团队共度时光并与社区中的其他人交流。

这些想法中有一些的进展较快，而有些可能最终不会变成好的主意，但我非常期待在接下来的几个月中继续探索它们。

当然，我们也会继续投入更多的 [组件示例](/plus/ui-blocks)、[模板](/plus/templates) 和 [Catalyst](/plus/ui-kit) 的改进！例如，我们非常渴望 **尽快为 Catalyst 设计一个组合框**，我们也即将开始为不使用 React 的人原型设计 **普通 HTML 版本的模板**。

因此，最终的 Tailwind Plus 是您所知道的高质量资源，但拥有全新的可能性和潜力。

做出这样的改变是一个大风险，但这是我想做多年的事情，我很高兴终于能够采取行动并发布它。

所以祝贺一个新时代的来临 — 迫不及待想为您构建一些令人惊叹的新东西。
